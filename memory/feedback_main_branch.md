Never commit directly to the main branch. Always follow this workflow:
1. Create a new feature branch
2. Commit changes to the branch
3. Create a pull request
4. Prompt for review before merging

**Why:** Maintains code review practices and keeps main branch stable.
**How to apply:** Any commit that would go to main must go through a branch + PR process instead.
